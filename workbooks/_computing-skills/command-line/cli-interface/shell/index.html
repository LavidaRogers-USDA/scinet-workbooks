### Nomenclature **CLI (Command Line Interface):** A text-based interface for interacting with the computer system by typing commands. **Terminal:** A tool or application that provides access to the CLI. **Shell:** The command interpreter that processes user commands entered into the CLI. **Bash:** A widely-used Unix shell known for its scripting and text processing capabilities. **PowerShell:** A Windows-based shell with object-oriented processing. ---- ## What is a Shell?
A shell is a **command interpreter** that processes user commands entered into the CLI and returns the output. It serves as a bridge between the user and the operating system, converting human-readable commands into a form the computer can understand and execute. Different operating systems use different shells or command interpreters; for example, Windows features PowerShell, while Unix-like systems use the **Unix shell**. Among these, **Bash shell** is the most commonly used, especially in high-performance computing (HPC) systems.
### How the shell works? The shell is the program running inside a [terminal](/glossary/#terminal) window, which provides a text-based interface for a user-computer interaction. Terminal emulators automatically load a default shell to process user commands and communicate with the underlying operating system. *For example:* - On Linux, `Bash` is often the default. - On Windows, `PowerShell` is standard.

Example

When you open a terminal window (see tutorial), you'll see a blinking cursor, waiting for you to type commands. The shell will try to interpret anything you enter there. 1. You type a command (e.g., `ls`). 2. The shell interprets the text-based command and communicates with the operating system. 3. The operating system executes the corresponding processes and sends the result back to the shell. 4. The shell displays the text-based result in the terminal window. ![shell command line](../assets/img/shell-command-line.gif)

---- ## **Unix shell:** `Bash`

What is the Unix Shell?

  • A command interpreter used to interact with Unix-like operating systems (e.g., Linux, macOS).
  • It allows users to control the system, run programs and automate tasks through text commands.
  • Bash (Bourne Again Shell) is the most widely used Unix shell, known for its scripting capabilities and broad support across systems, including SCINet HPC.
    • Bash is a widely used shell in high-performance computing (HPC) environments because most HPC systems are Linux-based. Learning Bash equips you with cross-platform skills that are essential for tasks ranging from everyday system management to advanced computational workflows in HPC environments, including SCINet clusters. Knowing Bash allows you to interact effectively with these systems, automate tasks and leverage powerful text-processing tools like `grep`, `awk` and `sed`. ### Basic terminology When working with Unix shells, understanding key terms is essential. These terms and concepts form the core of how shells make command-line interaction efficient and powerful. | term | definition | notes | |---------|------------|-------| | shell | A `command interpreter` that processes user commands and sends them to the OS for execution. | Examples include `Bash`, `Zsh`, `tcsh`, `ksh`. | | prompt | The text displayed by the shell indicating readiness to accept a command. | Often customizable and can include user, host, and current directory information. | | environment variable | A key-value pair used to store system or user-specific settings. | Examples include `PATH`, which defines the directories the shell searches for commands. | | `PATH` | An environment variable specifying directories where the shell looks for executable commands. | Example: Adding a directory to `PATH` allows running commands from that directory directly. | | command | A text-based instruction given to the shell to perform a specific action. | Common commands include `ls`, `cd` and `echo`. | | alias | A custom shortcut for a command or sequence of commands. | Example: `alias ll="ls -l"` creates a shortcut for `ls -l`. | | script | A file containing a series of commands that the shell can execute sequentially. | Typically used to automate repetitive tasks. | | shebang (`#!`) | A special line at the start of a script file specifying which `code system through a **prompt**, the starting point for entering commands. A **command** is a specific instruction to the shell, which can refer to a program, a built-in shell function or an **alias** - a custom shortcut for longer commands. For more complex tasks, you might use a **script**, a file containing a series of commands that the shell executes sequentially, automating repetitive workflows. To ensure scripts run with the correct interpreter, the **shebang** at the start of a script specifies the shell or program to execute it. Shells rely on **environment variables** to store important configuration data, like the directories listed in the **PATH**, which tells the shell where to look for executables. You can combine commands using **pipelines** to pass the output of one command as input to another, or use **redirection** to send the output of commands to files or other commands. If your workflow involves running multiple tasks, **job control** lets you manage processes in the background or foreground. ### Key features
    {% include accordion title="Command Interpretation:" class="outline " controls="bash-feature-1" %}
    The shell translates user commands into actions the operating system can execute. *Example: A single `ls` command interpreted by Bash retrieves and displays the list of files in the current directory.* ```bash ls # Output: file1.txt file2.csv script.sh ``` *Bash processes this input and communicates with the underlying system to retrieve the file list.*
    {% include accordion title="Interactive mode:" class="outline " controls="bash-feature-2" %}
    Running commands directly in real-time. Users type commands and the shell executes them immediately. ![shell command line](../assets/img/shell-command-line.gif)
    {% include accordion title="Batch mode:" class="outline " controls="bash-feature-3" %}
    Execute multiple commands or a script non-interactively for large-scale automation. *Example: Create a Bash script to run data analysis on multiple files. (Save the code below into the `analyze_data.sh` file.)* ```bash #!/bin/bash for file in *.csv; do python analyze_data.py "$file" > "${file%.csv}_results.txt" done ``` *Run the script in batch mode:* `bash analyze_data.sh`
    {% include accordion title="Pipeline and Redirection:" class="outline " controls="bash-feature-4" %}
    Combine commands in a sequence and redirect input/output to create efficient workflows. ```bash # combine commands with a pipe and redirect the output to a file ls | grep "file" > output.txt ```
    {% include accordion title="Scripting:" class="outline " controls="bash-feature-5" %}
    Automate repetitive tasks by running a series of commands for different inputs from a script file. ```bash # a simple Bash script for file in *.txt; do echo "Processing $file"; done ```
    {% include accordion title="Command history:" class="outline " controls="bash-feature-6" %}
    Access previously used commands for quick reuse and efficiency. *([see tutorial](/computing-skills/command-line/cli-interface/shell/commands/history))* *Example: Recall and reuse previous commands with the `up arrow` key or `history` command.*
    {% include accordion title="Customization:" class="outline " controls="bash-feature-7" %}
    Tailor the shell with aliases, custom prompts and configuration files. *([see tutorial](/computing-skills/command-line/cli-interface/shell/customization/index))* ```bash # create a shortcut for a custom combination of commands alias ll="ls -la" ```
    {% include accordion title="Job control:" class="outline " controls="bash-feature-8" %}
    Start, stop and manage running processes in the foreground or background. ```bash # run a process in the background and bring it back to the foreground sleep 100 & # Starts the process in the background jobs # Lists background jobs fg %1 # Brings job 1 back to the foreground ```
    {% include accordion title="Integration with many CLI tools:" class="outline " controls="bash-feature-9" %}
    Work seamlessly with powerful command-line tools like `grep`, `awk` and `sed`. ```bash # use grep or awk tools directly in the shell grep "error" *.txt # Searches for "error" in all text files awk -F ',' '{print $2}' data.csv # Extract the second column from a CSV file. ```
    {% include accordion title="Cross-platform interoperability:" class="outline " controls="bash-feature-10" %}
    Use shells like Bash across Linux, macOS and Windows (via WSL). ```bash # on Windows, install and launch WSL to use Bash wsl # then use standard Bash commands like: ls # Lists files in the current directory ```
    ### Why researcher should use Bash?
    By using Bash, you unlock the ability to interact efficiently with SCINet computational resources, automate workflows and handle data with precision, making it a vital skill for advancing your research.
    {% include accordion title="To access HPC systems and submit jobs" class="outline " controls="bash-usage-1" %}
    Bash is the default shell in most HPC environments, essential for running batch jobs, managing resources and automating computational workflows. Researchers can submit jobs to the scheduler, such as SLURM, using Bash scripts to define computational tasks, resource requirements and execution parameters. *Example: Submitting a job using a SLURM script:* ```bash #!/bin/bash #SBATCH --job-name=my_job # Job name #SBATCH --account=projectname # SCINet account #SBATCH --partition=atlas # Job queue type #SBATCH --nodes=1 # Number of nodes #SBATCH --ntasks=4 # Number of tasks #SBATCH --time=01:00:00 # Time limit (hh:mm:ss) #SBATCH --output=output.log # Output file module load python # load required module python my_script.py # execute the Python script ``` Submit the job with the `sbatch` command: `sbatch job_script.sh`
    {% include accordion title="To work across platforms" class="outline " controls="bash-usage-2" %}

    Bash runs on Linux, macOS and Windows (via WSL), providing a consistent interface and tools across any operating system.

    {% include accordion title="To customize commands for tasks" class="outline " controls="bash-usage-3" %}

    Researchers can define aliases, functions and reusable scripts, tailoring Bash to their specific project needs.

    {% include accordion title="To automate repetitive tasks" class="outline " controls="bash-usage-4" %}

    With its scripting capabilities, Bash simplifies repetitive tasks such as data preprocessing or file conversions and management.

    {% include accordion title="To process large datasets" class="outline " controls="bash-usage-5" %}

    Bash integrates with powerful command-line text-processing tools like `grep`, `awk` and `sed`, making it ideal for extracting, filtering and organizing data in bioinformatics, geospatial research and other fields.
