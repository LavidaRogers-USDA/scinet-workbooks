--- title: Introduction to Command Line Interface (CLI) description: "An interface for interacting with a computer by typing text commands instead of using a graphical interface (GUI)." index: 2 order: 1 tags: [command line interface] packages: level: 2 author: Aleksandra Badaczewska type: introduction --- ## Overview The [Command Line Interface](/glossary/#command-line-interface) (**CLI**) is a text-based way to interact with your computer, offering a more powerful and flexible alternative to using a graphical user interface (**GUI**), which relies on mouse-driven point-and-click navigation. To access the CLI, you use a **[terminal](/glossary/#terminal)**, which is the interface tool that provides a space for typing and viewing commands. By typing commands, you can perform tasks like managing files, running programs or configuring the system. Behind the scenes, a **[shell](/glossary/#unix-shell)** interprets these commands and communicates with the [operating system](/glossary/#operating-system) to execute them. Together, the CLI, terminal and shell form a powerful ecosystem for controlling your computer efficiently and performing your computational tasks with precision, speed, automation and a level of flexibility that is often unmatched by graphical user interfaces. ![GUI vs CLI](./assets/img/cli_vs_gui.png)

In this submodule

Discover the fundamentals of the [Command Line Interface (CLI)](/computing-skills/command-line/cli-interface/concepts/), including its key components: the [Terminal](/computing-skills/command-line/cli-interface/terminal/) *(interface tool)* and the [Unix Shell](/computing-skills/command-line/cli-interface/shell/), which interprets your text instructions *(i.e. commands)* entered in the [command line](/computing-skills/command-line/cli-interface/shell/commands/) input area.
