### Nomenclature **CLI (Command Line Interface):** A text-based interface for interacting with the computer system by typing commands. **Terminal:** A tool or application that provides access to the CLI. **Shell:** The command interpreter that processes user commands entered into the CLI. ---- ## What is a Terminal?
The terminal is a **software application** that provides a command line *(text-based)* interface (CLI) for entering and executing commands on a computer. By using predefined text commands (specific words or phrases that instruct the computer to perform tasks) you can navigate the file system, view or edit file content and run programs on your laptop or other computing machines, including high-performance computing (HPC) clusters.
On any platform, you can open the Terminal (or a terminal emulator) from the graphical interface, just like any other application and it will appear as a window on your screen. By default, terminals often use a well-known dark theme with light text for better readability. ![terminal application - CLI tool](../assets/img/terminal-mac.png) *(Terminal can be launched from the **Applications** on a macOS; terminal window looks similar across platforms)* ### Interface The terminal interface is a simple, **text-based window** where you can type and execute commands. It typically displays a [prompt](/glossary/#prompt), which indicates that the terminal is ready to receive input. The prompt often includes information like the username, hostname and current directory. *(learn more from the Terminology Explained/Terminal interface elements)*
After you type a command and press `Enter`, the terminal executes it and displays the output or an error message directly below. The interface is minimalist, focusing entirely on text interaction without graphical elements. *(learn more from the Terminal customization)* ![terminal application - CLI tool](../assets/img/terminal-interface.gif) ### Key features * **Input and Output:** Users type text commands (input), and the terminal displays responses or results (output). *(see terms)* * **Customization:** Terminals can be customized with themes, fonts and shortcuts for better usability. *(see tutorial)* * **Command Help:** Learn about commands directly in the terminal by using built-in help tools. *(see tutorial)* * **History and editing:** Modern terminals support command history *(see tutorial)* and inline editing *(see tutorial)* for efficiency. * **Remote Access:** Terminals enable users to connect to remote resources, such as SCINet HPC systems. *(see tutorial)* ### Basic terminology When working in a terminal, understanding basic terminology is essential. It not only helps you use the terminal more effectively but is also crucial when seeking assistance or describing an issue accurately. | term | definition | notes | |---------------------|------------|-------| | Terminal (emulator) | A program that provides a text-based interface to interact with the operating system. | Examples include GNOME Terminal, iTerm or Windows Terminal. | | Welcome Message | The startup message when logging into a remote machine. | It usually tells you about the architecture of available resources and rules for the users. | | Prompt | The text next to where you type your commands. | Prompts can be modified to include additional information like hostname or current folder location. | | Command-Line | The specific place within the terminal interface where commands are entered. | Usually indicated by a blinking cursor next to the **prompt**, where users type commands to execute. | | Command | The program or script you are trying to run. | Built-in commands and aliases are called by name, while custom Bash scripts stored in files can be executed with 'dot syntax' : `. ~/path/script.sh`. | Option or Argument | Added to a command to modify the output. | There is always a space between a command and the argument. | | Standard Out | The text result of a command. | Use `> filename1` to redirect output into the file. Use `>> filename1` to append output to the end of a file. | | Standard Error | The error message of a failed command. | Use `2> filename2` to redirect error message into the file. Use `2>> filename2` to append error message to the end of a file. | ![terminal components](../assets/img/terminal-explained.png) When you open a **terminal** (emulator, CLI tool), you may first see a **welcome message** providing details like the last login date, system architecture, and user guidelines set by administrators. Below it, the **prompt** appears. This line is where you interact with the system. The **command-line** is the space where you type commands: names of programs, built-in shell processes, aliases or executable scripts. Pressing `Enter` executes these commands, instructing the system to perform a specific task. Many commands accept **arguments**, which are additional inputs or options that modify their behavior or output. Once executed, commands typically produce text output. This information is sent to **standard output** (stdout), usually displayed as text in the terminal. If something goes wrong, error messages are sent to **standard error** (stderr). Both stdout and stderr can be redirected to files for later use, allowing efficient handling of data and error logs. ### Why Terminal instead of a GUI? The GUI provides a visual, user-friendly interface that is intuitive for basic tasks like browsing and managing individual files, while the CLI accessed via Terminal excels in efficiency, especially for complex or repetitive tasks, offering precision and flexibility. ![CLI vs GUI](../assets/img/cli_vs_gui.png) *(Graphical User Interface shows a file manager with icons and folders; Command Line Interface displays the same folder's contents in a text-based terminal using the `ls` command)* The terminal's CLI provides unique and powerful capabilities that often surpass GUIs for both research-focused and everyday tasks, offering greater flexibility for repetitive steps and advanced computing needs alike.

Example

Renaming thousands of image files manually in a GUI would be a time-consuming and error-prone task. With a single command in the terminal, such as a loop or a batch rename utility, you can quickly and accurately rename all files based on a pattern (e.g., adding a prefix or changing extensions), saving significant time and effort.
*For example, this command renames all `.jpg` files present in a current directory (i.e., folder) in seconds.* ``` for file in *.jpg; do mv "$file" "new_$file"; done ```
**Here a few advantages of using CLI tool to support your research:**
{% include accordion title="Efficiency" class="outline " controls="terminal-1" %}

Perform repetitive tasks or batch operations with fewer steps using commands instead of manual point-and-click interactions within a graphical interface. *(like in the example shown above)*

{% include accordion title="Automation through scripting" class="outline " controls="terminal-2" %}

Terminals allow you to write and execute scripts to automate repetitive tasks and streamline complex workflows.

{% include accordion title="Access to advanced tools" class="outline " controls="terminal-3" %}

Certain powerful tools and utilities, such as text-processing programs (`awk`, `sed`) or high-performance computing job schedulers (e.g., **Slurm**), are only available in the CLI, enabling tasks that GUIs cannot handle.

{% include accordion title="Remote access" class="outline " controls="terminal-4" %}

The terminal allows users to securely connect to and manage remote systems using protocols like SSH or browser-based tools like Open OnDemand (OOD), providing efficient interaction with servers, cloud environments or HPC resources from anywhere.

{% include accordion title="Remote session persistence" class="outline " controls="terminal-5" %}

In-terminal tools like `tmux` or `screen` allow users to maintain and reattach to terminal sessions, even after disconnecting from a remote system.

{% include accordion title="Pipelines and redirection" class="outline " controls="terminal-6" %}

You can chain commands together using pipelines (`|`) and redirect input/output to or from files, enabling advanced data processing and customization.

{% include accordion title="Multitasking and process management" class="outline " controls="terminal-7" %}

Terminals support running multiple processes simultaneously, including background processes, and provide commands like `top`, `ps` and `kill` to monitor and manage them.

{% include accordion title="Integration with version control systems" class="outline " controls="terminal-8" %}

Terminals provide seamless interaction with version control tools like Git, essential for collaborative software development and research.

{% include accordion title="Cross-platform consistency" class="outline " controls="terminal-9" %}

While terminal apps vary by platform, the core command-line experience is consistent, making it easier to transfer skills across systems.

---- ## Examples of Terminal emulators Each operating system has its own set of terminal applications (or terminal emulators), but they all share common behaviors and features, such as text-based command input, output display and support for various **CLI environments**. These terminal apps allow users to interact with the system's CLI, manage files, run commands and connect to remote systems such as SCINet HPC.
CLI environments refer to the underlying **shells that interpret and execute the commands** entered in a terminal. These environments, like `Bash`, `Zsh` or `PowerShell`, define the syntax and functionality available for command-line interaction with the specific operating system.
In this lesson module, we focus on CLI tools ([terminal apps](/glossary/#terminal)) that support the **[Unix shell](/glossary/#unix-shell)**, a widely-used CLI environment ([command interpreter](/glossary/#cli-environment)). It is the standard for accessing and operating on HPC systems (including SCINet clusters), making it an essential tool for advancing the computational aspects of your research. Learn more in the lesson module What is a Unix Shell? The Command Interpreter for the CLI.
### for Windows Windows provides several terminal-like applications, each with its own features and capabilities. | CLI tool | description | Features/Limitations | pre-installed | |----------|-------------|----------------------|---------------| | Command Prompt | The classic Windows CLI for running batch scripts and **native Windows commands**. | Limited to Windows commands; no native Unix shell support. | Yes | | PowerShell | An advanced scripting environment for automation and system management in Windows. | Supports scripting; **requires WSL for Unix shell**. | Yes | | Windows Terminal | A modern terminal application that supports tabs, customization and integration with WSL. | Fully customizable; supports PowerShell, Command Prompt and WSL for Unix shell. | Yes/No |

SSH to SCINet clusters on Windows

Windows provides a way to access remote SCINet HPC clusters using SSH, and **SSH functionality is directly available in PowerShell** (starting with Windows 10 version 1809 and later) as well as in Command Prompt if `OpenSSH` is installed. If your local computing machine meets these requirements, follow the instructions provided in the tutorial Command-line access to SCINet HPC.

Run Bash scripts on Windows

Windows CLI tools, such as `Command Prompt` and `PowerShell`, natively interpret only Windows-specific syntax and commands, making them incompatible with Unix-style commands and scripts *(used on SCINet clusters)*. **PowerShell alone cannot directly execute Bash scripts.**
To run Unix/Bash scripts on Windows machine, you must use Windows Subsystem for Linux (WSL) to create a Unix-like environment. **WSL provides full compatibility with Unix tools and scripting.**

#### Recommended Tool for Unix Shell: `PowerShell` with WSL To get a Unix shell (e.g., Bash) on Windows with minimal effort, you can install Windows Subsystem for Linux (WSL).

Scroll through the list of applications to find `Windows Terminal` or `PowerShell`, typically at the top.
Click to launch Terminal window. *The **Windows Terminal** app allows you to have multiple tabs in a single window.*
